Blueprints

Blueprints are graphs of nodes properly connected to produce some result or effect.
Every blueprint has at least two nodes:

  • Entry point where execution begins.
  • Return where execution ends.
Example: a very simple blueprint that calculates the area of a circle given the radius as input:
Area = π * Radius * Radius

A blueprint might have an undefined number of input parameter of any type and zero or one return value, just like a Java function.

Every program has two predefined blueprints: Main and Events.

The Main blueprint

The Main blueprint is where the execution begins when a program is executed. There is a single input parameter which is an array of String, and doesn't return anything.
This blueprint is something like the main method in a Java program.

The Events blueprint

The Events blueprint doesn't have a single entry point but has more nodes that are triggered when some events occur:
  • Event Begin is triggered when program starts
  • Event Exception is triggered when a critical exception occurs
  • Event End is triggered when program ends
